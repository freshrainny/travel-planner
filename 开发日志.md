# AI 旅行规划师 · 开发日志

> 记录从 0 到 1 构建这个工具的完整过程，包含每个功能的设计思路和技术决策。

---

## 起点：一次对话

**2026年5月** — 从一个问题开始：「旅游过程中有哪些 AI 可以发挥作用的地方？」

讨论了 AI 在行前规划、出行途中、美食推荐、旅行记录等环节的应用后，话题落到了具体工具上。

> 用户试用了 TripGen 之后的反馈：  
> "他生成的地点不太合理，比如威尼斯其实只需要1-2天就能玩完，并且没有米兰、多洛米蒂、那不勒斯这些城市。我想要可以输入时间和地点，安排每一天干什么，并且**可以修改**的旅行规划工具。"

这句话定义了这个项目的核心价值：**可编辑的 AI 行程规划**。

## 功能迭代记录

### v0.1 — 基础框架
**核心需求：**
- 输入目的地、出发日期、天数、旅行偏好
- AI 生成每天行程，包含景点、时间安排
- 支持手动编辑（修改、添加、删除景点）
- 拖拽排序

**技术选型：**
- 纯 HTML + CSS + Vanilla JS，单文件，无需构建工具
- SortableJS 实现拖拽
- 支持 Claude、GPT、DeepSeek 多模型（统一 `callAI` 接口）

**解决的关键问题：**  
Token 超限 → 引入**两阶段生成**：先生成行程框架（骨架），再按需懒加载每天的详细内容。避免一次性生成10天行程导致超限或内容简单化。

---

### v0.2 — AI 模型支持完善

**背景：** 用户问「gpt4o不是死了吗」「gpt5.5都有了」「我想用deepseek api怎么用」

**方案：**
- 支持 Anthropic（Claude）和 OpenAI 兼容格式（GPT-5、DeepSeek）
- 可自定义 API Endpoint，兼容任意 OpenAI 格式接口
- 加入 `repairJSON` 函数 + `json_object` 响应格式，修复 AI 输出的 JSON 解析问题

---

**改动：**

### v0.4 — 地图可视化 + 交通路线

**用户需求：**  
"怎么修改具体景点顺序和时间？怎么增加景点之间交通路线和交通方式？可以做成地图式展示吗？"

**实现：**
- 集成 **Leaflet.js** 地图，景点自动按顺序标注编号
- AI 在加载详情时同步生成经纬度坐标和 `transport_to_next`（到下一个景点的交通方式）
- 地图支持按天筛选，切换查看每天路线
- 左侧行程 + 右侧地图的分栏布局（标签页切换）

**地图语言问题：**  
OpenStreetMap 默认显示当地语言（如日文、韩文）。  
解决：换用 **CartoDB Voyager** 底图，全程英文标注。

```javascript
L.tileLayer('https://{s}.basemaps.cartocdn.com/rastertiles/voyager/{z}/{x}/{y}{r}.png', ...)
```

### v0.5 — 住宿选择

**需求：** 为每一天设置住宿点，在地图上单独标注

**实现：**

### v0.6 — 行程分享

**需求：** "这个网页可以分享给别人使用吗？"

**方案：** URL Hash 编码（纯前端，零后端）  
行程数据 → JSON → Base64 → 拼入 URL `#data=...`

```javascript
const encoded = btoa(unescape(encodeURIComponent(JSON.stringify(shareData))));
const url = `${location.href.split('#')[0]}#data=${encoded}`;
```

**优点：** 不需要服务器、不需要数据库，朋友打开链接即可查看完整行程  
**缺点：** 行程越长，链接越长

---

### v0.7 — 景点 AI 攻略弹窗 🔍

**需求：** "能不能给景点加详情和攻略搜索？景点说明、买票方式、注意事项、吃饭推荐周边餐厅等"

**实现：** 每个活动卡片悬浮时出现 🔍 按钮，点击弹出四标签页攻略弹窗：

| 标签 | 内容 |
|------|------|
| 📋 概览 | 景点介绍、亮点、最佳时间、建议时长、人流 |
| 🎫 门票与时间 | 票价、购票方式、开放时间、闭馆日、免票政策 |
| ⚠️ 注意事项 | 4+ 条实用提示 |
| 🍜 周边美食 | 3-5 家周边餐厅，含距离、人均、特色标签 |

攻略由 AI 实时生成，支持 🔄 重新生成。

### v0.9 — localStorage 自动保存 + 多行程管理

**背景：** "只是一个 HTML，能做 localStorage 和多行程管理吗？"

**localStorage 自动保存：**
- `renderTrip()` 调用后自动触发保存（800ms 防抖）
- API Key、模型选择 也持久化保存
- 刷新页面行程不丢失
- 优先级：URL Hash 分享 > localStorage 恢复

**多行程管理（📚 我的行程 抽屉）：**
- 右侧滑入抽屉面板
- 支持保存、切换、重命名、删除多个行程
- 每张行程卡片显示目的地、天数、最后更新时间
- 当前激活的行程用蓝色「当前」标签标注

## 架构总结

```
用户浏览器
├── index.html（单文件应用）
│   ├── CSS（内联，约500行）
│   ├── HTML（结构 + 所有弹窗）
│   └── JavaScript（约2000行）
│       ├── callAI()            → 统一 AI 调用接口
│       ├── generateTrip()      → 骨架生成
│       ├── loadDayDetail()     → 按天懒加载详情
│       ├── renderTrip()        → 行程渲染 + 自动保存
│       ├── initMap/refreshMap  → Leaflet 地图
│       ├── openAttractionGuide() → 景点攻略 AI 生成
│       ├── saveCurrentTrip()   → 多行程管理
│       └── loadFromHash()      → 分享链接解析
│
├── GitHub Pages 托管
└── CartoDB 地图瓦片（英文）
```

**数据流：**
```
用户输入 → AI API（Claude/GPT/DeepSeek）
                ↓
         JSON 行程数据 → localStorage 自动保存
                ↓                    ↓
         Leaflet 地图渲染    URL Hash 分享链接
```

## 技术债 & 可改进方向

| 问题 | 优先级 | 解决方案 |
|------|--------|---------|
| 分享链接过长 | 中 | 用 jsonbin.io 等免费存储，生成短链 |
| 数据不跨设备 | 中 | 加后端（Supabase/Firebase 免费额度） |
| AI 攻略信息时效性 | 低 | 标注生成时间，提醒以官方为准 |
| 地图坐标精度 | 低 | AI 生成坐标偶有偏差，可接入 Geocoding API 校正 |
| 移动端适配 | 低 | 目前分栏布局在小屏有压缩，需要优化 |
